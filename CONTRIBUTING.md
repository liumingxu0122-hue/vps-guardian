# Contributing

VPS Guardian welcomes narrowly scoped fixes and design discussions suitable for an early alpha.

1. Open an issue before a large behavioral or schema change.
2. Fork the repository and create a focused branch.
3. Never commit credentials, infrastructure identifiers, private evidence, databases, logs, or generated build trees.
4. Run Python lint, types and tests; Go format, vet and tests; and Web typecheck, unit tests and build.
5. Add tests for changed security boundaries and operator-visible behavior.
6. Submit a pull request describing risk, rollback, and validation.

Contributions are licensed under Apache-2.0. Third-party code must retain its original notices and use a compatible license.
